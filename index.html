Investigacion de JavaScript

Preguntas

  1. ¿Qué versiones de JavaScript hay en el mercado?
  2. ¿Cómo se integra JavaScript a HTML?
  3. ¿Cómo es el manejo de las variables, tipos de datos y expresiones regulares en JS?
  4. ¿Cómo son todas las formas de declaración de funciones en JS?
  5. ¿Cuáles son los operadores y conversión entre tipos?
  6. ¿Cuáles son las estructura de control de JS?
  7. ¿Qué es el objeto dom en JS? Y ¿Cómo trabaja?
  8. ¿Qué función tiene en JS el Object Map?
  9. ¿Cuáles son todos los eventos que trabaja JS?
  10. Con toda la investigación que generaron deben de generar un formulario que realice las siguientes funciones.
    • ¿Qué obtenga los datos del input?
    • ¿Qué verifique los datos que se ingresan en el input?
    • ¿Qué el formulario presente una calculo de sumatoria para validar ¿Qué no sos un robot”?

Respuestas

  1. Antes de mencionar las versiones de JavaScript es importante aclarar que JavaScript fue adoptado como un estándar de la ECMA, para luego recibir el nombre de ECMAScript. Por ende, cuando hablemos de las versiones de ECMAScript también estaremos haciendo referencia a las versiones de JavaScript.

  2. JavaScript se puede integrar de 3 formas en el archivo HTML:
    • Por la etiqueta <script>dentro de la etiqueta <head>
    • Por la etiqueta <script src ""> dentro de la etiqueta <head>
    • Por la etiqueta <script> dentro de la etiqueta <body>

  3. El manejo de variables en JS se hace mediante let, var y const. let y var se utilizan para definirle los valores a una variable que en un futuro puede cambiar. Mientras que const se utiliza para darle un valor a una variable que no va a cambiar. En cuanto a los tipos de datos, JS pertenece a los lenguajes dinámicos, por ende, no hace falta definir el tipo de dato de una variable de forma manual, el propio lenguaje tiene la capacidad de identificar el tipo de dato de la variable por el valor que le demos. Sin embargo, nunca está de más mencionar los tipos de datos que tenemos en JS:

    • Number
    • String
    • BigInt
    • Boolean
    • Undefined
    • Function
    • Null
    • Symbol
    • Object

    Por último, las expresiones regulares en JS sirven para buscar, capturar o reemplazar texto mediante la utilización de patrones. En JS el manejo de expresiones regulares se hace mediante caracteres y símbolos. Cuando haya una expresión regular en algún código JS nos daremos cuenta porque estarán definidos por RegExp o RegEx.

  4. Hay muchas formas de crear funciones en JavaScript, entre ellas se destacan:

    • Funciones por declaracion
    • Funciones de expresion
    • Funciones como objetos
    • Funciones anonimas
    • Funciones callbacks
    • Funciones autoejecutables
    • Funciones flecha (arrows functions)

  5. Los operadores de conversión son operadores que convierten el valor de una variable a valor correcto o también pueden convertir el valor a un tipo de dato deseado.

    • ToString (convierte el valor a tipo texto)
    • ToNumber (convierte el valor a tipo numerico)
    • ToBoolean (convierte el valor a tipo booleano)

  6. Las estructuras de control en JS son similares a las de otros lenguajes de programación como Java, C, entre otros. Sin embargo, en algunas estructuras encontraremos algunas diferencias.

    • Condicionales: Estas estructuras son las que utilizamos para decidir o condicionar el código. En JS los condicionales se hacen de distintas maneras:
      • If: condición simple
      • If/else: condición con una alternativa.
      • (?:) : condición if/else pero abreviada con el operador ternario.
      • Switch: estructura con una seguidilla de pasos, similar al if/else anidado.
    • Bucles y repetitivas: son estructuras de control para automatizar el código y no tengamos q repetir manualmente ciertas sentencias:
      • While: bucle simple que funciona mediante la condición “mientras”.
      • For: bucle similar al while.
      • Do…while: bucle que funciona mediante la condición “hasta”.
      • For…in: bucle que funciona sobre posiciones en un array.
      • For…of: bucle que funciona sobre elementos de un array.

  7. El objeto DOM en JS hace referencia al documento HTML, que desde JavaScript podremos modificarlo de manera dinámica, agregando o eliminando etiquetas, cambiando el contenido, aplicar nuevas clases o modificar las que ya están establecidas, entre muchas modificaciones mas. El DOM trabaja mediante el objeto document, que representa el conjunto de etiquetas en el archivo HTML. En el interior de este objeto pueden existir varios tipos de elementos como element o node.

    • Element: es la representación de una etiqueta HTMLElement. Por ej. HTMLDivElement, HTMLSpanElement, HTMLImageElement, entre otras.
    • Node: es la representación de un nodo de texto.

    En JavaScript también se pueden buscar elementos en el DOM:

    • .getElementById( ): busca el elemento HTML con el id que pongamos dentro de los paréntesis.
    • .getElementsByClassName( ): busca un elemento HTML con la clase que pongamos dentro de los paréntesis. Hay que aclarar que en el caso de que existan varios elementos con la misma clase o dentro de la misma clase, nos devolverá un array.
    • .getElementsByName( ): busca los elementos por el atributo name.

    Hay otros dos métodos nuevos para buscar elementos, pero van destinados al CSS:

    • .querySelector( ): buscara el primer elemento que coincida con el selector que pongamos dentro de los paréntesis.
    • .querySelectorAll( ): buscara todos los elementos que coincidan con el selector que pongamos dentro de los paréntesis. Devolverá un array.

  8. El objeto mapa en JS es una estructura de dato que funciona como un mapa, guardando distintos valores a través de distintas claves para identificarlos. Este tipo de estructura la encontraremos denominada como clave-valor. En JS el objeto mapa nos ayudara a gestionar un conjunto de datos de la manera que queramos mediante métodos o propiedades.

    • .set(key, value): establece o modifica la clave key con el valor value.
    • .has(key): comprueba si key ya existe en el mapa.
    • .get(key): obtiene el valor de la clave key en el mapa.
    • .delete(key): elimina el valor con la clave key en el mapa.
    • .clear( ): vacía el mapa completamente.
